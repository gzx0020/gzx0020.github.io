Score-based diffusion

Posted on 2026-01-28 Edited on 2026-02-01 In 深度学习

1.分数Score

score的数学表达为 ∇x̃log pσ(x̃)

其中pσ(x̃)是加噪后的概率密度函数，来自卷积 pσ(x̃) = ∫pdata(x)𝒩(x̃; x, σ2I)dx ，即真实数据分布和扰动核（perturbation kernel）的卷积，等价于对数据分布做一次高斯平滑，原因：与高斯核卷积之后得到一个处处可导的密度函数，保证梯度存在。

但是由于以上形式的score中包含原始数据分布，所以根据一系列推导，得到 $$\nabla_{\tilde{x}}\log p_\sigma(\tilde{x})=\mathbb{E}{\left[\left.\frac{x-\tilde{x}}{\sigma^2}\right|\tilde{x}\right]}$$ ，即 score = 𝔼[denoising direction ∣ x̃] ，其中denoising direction为 $\nabla_{\tilde{x}}\log p_\sigma(\tilde{x}\mid x)=\frac{x-\tilde{x}}{\sigma^2}$

总的来说，score是一个向量场，指向概率更大的方向

2.SCORE-BASED GENERATIVE MODELING WITH SDEs

2.1. 引入一个前向SDE：

dx = f(x, t)dt + g(t)dw,

其中 x(t) 是数据，f(x, t) 是漂移项（drift）, g(t) 是噪声强度，dw 是标准wiener过程（高斯噪声）

2.2. 给定一个前向的SDE，它的逆过程仍然是一个SDE

形式是$$d\mathbf{x}=\begin{bmatrix}f(\mathbf{x},t)-g^2(t)\nabla_\mathbf{x}\log p_t(\mathbf{x})\end{bmatrix}dt+g(t)d\bar{\mathbf{w}}$$

注意反向SDE的漂移项包含score的表达式。

至此可以总结：Score-based generative modeling = 从 pT(x) ，根据学到的score, 数值求解反向SDE

2.3. 接着介绍了VE/VP两两种SDE形式

VE SDE 对应连续时间的去噪 score matching，VP SDE 则统一了 DDPM 的扩散过程。

3.Solving the Reverse SDE

需要把反向SDE dx = [f(x, t) − g2(t)∇xlog pt(x)]dt + g(t)dw̄ 变成可以在电脑上一步步求解的采样算法。

3.1.用 Euler–Maruyama 方法离散 SDE

Euler–Maruyama 方法：对于一般的SDE dx = μ(x, t)dt + σ(t)dw ，将其离散为 $\mathbf{x}_{k-1}=\mathbf{x}_k+\mu(\mathbf{x}_k,t_k)\Delta t+\sigma(t_k)\sqrt{\Delta t}z_k$ ，即漂移项 × dt + 随机项 × √dt × 高斯噪声

对于反向SDE μ(x, t) = f(x, t) − g2(t)sθ(x, t)

将其离散为$$\mathbf{x}_{k-1}=\mathbf{x}_k+\left[f(\mathbf{x}_k,t_k)-g^2(t_k)s_\theta(\mathbf{x}_k,t_k)\right]\Delta t+g(t_k)\sqrt{\Delta t}\left.z_k\right.$$

注：这里的μ(x, t)是漂移项（请对照前面反向SDE的公式，只有漂移项才跟score有关系，随即项与score无关），其中的score换成网络近似sθ(x, t)

3.2.Predictor–Corrector (PC) 方法

Predictor：就是刚才离散话的步骤

Corrector：用score修正，即在固定时间 𝑡，沿 score 做 MCMC 采样

$$x\leftarrow x+\epsilon s_\theta(x,t_k)+\sqrt{2\epsilon}z,\quad z\sim\mathcal{N}(0,I)$$

PC sampler 的完整算法结构：

  1. 初始化 xT ∼ 𝒩(0, σ2(T)I)

  2. for k = T → 1

  • Predictor step（Euler–Maruyama）
  • Corrector step（多步）
  1. 输出x0

3.3 Probability Flow ODE

把随机项去掉，得到Probability Flow ODE，$$\frac{d\mathbf{x}}{dt}=f(\mathbf{x},t)-\frac{1}{2}g^2(t)\nabla_\mathbf{x}\log p_t(\mathbf{x})$$ ，这个ODE与SDE不同路径但同分布，可以做 likelihood 计算

4.Conditional Generation

为了能够生成我们想要的图片，引入condition y ，则score变成条件 score ∇xlog pt(x ∣ y) ，我们真正想采样的是条件y下的数据分布 p(x ∣ y) ，将其用贝叶斯公式再求对数，最后求梯度得 ∇xlog p(x ∣ y) = ∇xlog p(x) + ∇xlog p(y ∣ x)

将前面反向SDE中的score换成条件score得 dx = [f(x, t) − g2(t)(∇xlog pt(x) + ∇xlog pt(y ∣ x))]dt + g(t)dW̄ 其中 drift = f(x, t) − g2(t)∇xlog pt(x) − g2(t)∇xlog pt(y ∣ x) 为原来的反向漂移项-控制项∇xlog pt(y ∣ x)

实践中常引入一个控制强度λ 表示条件的强度

∇xlog pt(x ∣ y) = ∇xlog pt(x) + λ∇xlog pt(y ∣ x)

自然，Probability Flow ODE形式将变成

$$\frac{dx}{dt}=f(x,t)-\frac{1}{2}g^2(t)\left(\nabla_x\log p_t(x)+\nabla_x\log p_t(y\mid x)\right)$$

FastKV

Posted on 2025-10-03 In 论文阅读笔记

FastKV 论文阅读笔记

1. 论文在解决什么问题？

大语言模型（LLM）处理长文本（比如128K tokens）时，有一个很大的瓶颈：KV Cache。 - KV Cache是什么？ 为了在生成每个新token时不用重新计算所有历史token的信息，Transformer的Attention层会把之前所有token的Key和Value缓存下来。(可以读我之前写的注意力机制) - 问题在哪？ 文本越长，KV Cache，导致： 1. 内存不够用：例如，处理128K tokens的文本，KV Cache可能比模型本身还占内存。 2. 速度变慢：尤其是在“预填充”阶段（Prefill Stage，即处理用户输入的整个提示词），需要为所有输入token计算并存储KV Cache，计算量和延迟很高。TTFT（首token时间）是影响用户体验的关键指标。

现有的KV Cache压缩方法，比如SnapKV，主要在生成阶段节省内存，但对预填充阶段的加速没什么帮助。而另一个方法GemFilter虽然能加速预填充，但因为它直接删掉了输入提示词中的部分token，导致模型丢失了全局上下文信息，精度下降。

所以，这篇论文的目标就是：找到一个既能大幅加速预填充阶段（降低TTFT）、又能保持模型精度的KV Cache压缩方法。 - 补充：什么是TTFT？

TTFT 的全称是 Time To First Token，指的是从你按下送，把完整的Prompt交给大模型开始，到大模型生成的第一个token为止所花费的时间。

2. 核心方法：FastKV 与 Token-Selective Propagation (TSP)

FastKV 的核心方法是 Token-Selective Propagation (TSP) 机制。

核心思想

  1. 早期层（TSP层之前）：让模型像正常一样，完整地处理所有输入token。这样，每个token都经过了多层Attention的“信息融合”，已经包含了丰富的上下文信息。
  2. 在某个中间层（TSP层）：从所有token中，筛选出一小部分“最重要”的token。
  3. 后期层（TSP层之后）：只对这一小部分筛选出来的token进行后续计算。因为token数量大大减少，后续层的计算量也大幅降低，从而实现了加速。

$S_i^{l,g}=\frac{1}{H_G}\sum_{h=h_g}^{h_g+H_G}S_i^{j,h}$

( S_i^{l,g} ) 第 l 层、第 g 个注意力Attention Head Group中，第 i 个 token 的组重要性得分。这是该公式的计算结果，代表一个Attention Head Group对某个 token 的注意力得分。

( H_G ) 每个注意力Attention Head Group中包含的 Attention Head 数量。这是一个预设的超参数，用于控制组的规模。

( h_g ) 第 g 个组的起始 Attention Head 索引。例如，如果每组有8个头，那么第一组 ( h_g = 0 )，第二组 ( h_g = 8 )，以此类推。

( h_g + H_G ) 第 g 个组的结束 Attention Head 索引。即对一个完整组内所有头进行遍历。

( S_i^{j,h} ) 第 j 层、第 h 个 Attention Head 中，第 i 个 token 的重要性得分。

  • 这里插入一个和SnapKV的对比 SnapKV 为每个头单独计算和选择， 每个头选自己的token，不同头保留的token集合可能完全不同

$S_i^{l,h}=\frac{1}{2w_p+1}\sum_{m=-w_p}^{w_p}\sum_{n=0}^{N_\mathrm{obs}}Att_l[h,N_I-n,i+m]$

FastKV 按组统一计算和选择， 整个组共享同一套token，组内所有头保留相同的token集合

SnapKV为每个头独立选择token，而FastKV按头组统一选择token，并且通过TSP机制实现了预填充阶段的加速

TSP 如何选择重要token？

利用Attention机制本身来评估token的重要性。

  1. 计算注意力分数：在选定的TSP层，对于该层的每一个Attention Head，我们都能得到一个注意力分数矩阵 Att_l。
  2. 计算token重要性得分：为了得到一个统一的、代表整个层观点的分数，论文将所有Attention Head的分数进行平均： [ S_{i}^{TSPlayer} = {h=0}^{H-1} S{i}^{TSPlayer, h} ]
    • ( S_{i}^{TSPlayer} )：第 i 个token在TSP层的重要性得分。
    • ( H )：该层的Attention Head总数。
    • ( S_{i}^{TSPlayer, h} )：第 i 个token在第 h 个Head中的重要性得分（这个得分可以理解为对注意力分数进行滑窗平均得到的）。
  3. 选择Top-K：根据 ( S_{i}^{TSPlayer} ) 得分，选择得分最高的K个token。这个K就是 TSP长度（TSP Length）。论文发现TSP长度设为2048是个比较好的平衡点。
  • 注意一个时间节点：

也就是说TSP操作发生的时间点是“after the MLP block”，在 TSP 层，模型首先像正常一样，对所有输入 token 执行了 自注意力计算 和 MLP 计算。在经过了 MLP 块之后得到了该层最终的、经过充分信息处理的隐藏状态。此时，FastKV 才使用之前选出的重要 token 的索引，从这个完整的隐藏状态矩阵中，索引出那些重要 token 对应的隐藏状态。这些被挑出来的隐藏状态，会被传递给下一层。由于 token 数量大大减少，下一层的计算量也随之大幅降低。

两个关键参数的解耦

TSP长度 和 KV Cache预算（KV Budget） 是两个独立的参数。这样的涉及便于实际应用中平衡速度和精度。

  • TSP长度：决定有多少个token会参与TSP层之后的计算，影响预填充阶段的计算速度（TTFT）。
  • KV Budget：决定最终有多少个token的KV值会被存入Cache，影响生成阶段的内存占用和吞吐量（Throughput）。

例如，可以设置 TSP长度=2048，KV Budget=512。这意味着： - 预填充时，后期层只处理2048个token，计算量小，TTFT低。 - 生成时，KV Cache里只存了512个token的Key和Value，内存占用小，吞吐量高。

5. 一些疑问和思考

5.1 （续上面对SnapKV和FastKV的对比）如果有超长代码生成等任务（或者其他细粒度依赖的任务），FastKV采用的是按组进行token选择，而不是为每个注意力头单独选择token，假设在代码生成任务中，有一个注意力头专门负责结构体的定义，另一个头负责api的调用。这两个头关注的内容不同，因此它们认为重要的token也可能不同。如果这两个头被分在同一个组，那么组的重要性得分是这两个头得分的平均。这可能导致一些对其中一个头非常重要但对另一个头不重要的token被平均掉，从而不被选择。而在SnapKV中，每个头都可以保留自己最重要的token，因此不会出现这个问题。

创建信号量

信号量本质上是一个特殊的 RTOS 内核对象，用于任务之间或任务与中断之间的 同步 或 资源管理。

可以理解为定时器，但是装载的不是数字而是信号数目，当信号数目大于0时任务可以进行但要获取一个信号数目（即信号-1），信号数目=0时任务阻塞，等到有信号再唤醒。

二值信号量 (Binary Semaphore)

只能取值 0 或 1，相当于开关；最常用来让一个任务等待某个事件发生，示例如下

计数信号量 (Counting Semaphore)

计数范围大于 1，可用于控制对某类资源的访问次数

简单理解，可以这样用：有个事件A释放信号，设置Count（信号量最大数目）为n,当A执行满n次后开始执行任务B

互斥量可以这样理解：二值信号量 + 优先级

使用方法与信号量相同

普通互斥量

只有一个任务能持有（值为 0 时其他任务阻塞），不能由另一个任务释放（必须持有者释放），同一任务如果再次请求同一 Mutex → 死锁（因为它已经占用，自己还在等自己释放），即只能请求一次。

递归互斥量

允许同一个任务多次获取同一把锁，每次获取都要匹配一次释放。必须释放和获取次数匹配，计数归零时才真正释放锁。

2.决策树的构建标准

2.1 信息增益

2.1.1 信息熵

$H\left(S\right)=-\sum_{i=1}^{c}p_{i}\log_{2}\left(p_{i}\right)$

其中，S 表示样本集，C 表示样本集合中类别个数（只含有正负样本，则 C=2），pᵢ表示第 i 个类的概率，（pᵢ可由类别 i 中含有样本的个数除以总样本数得到）

取值范围：0 ≤ H(X) ≤ log2n 即所有结果等概率时熵最大

直观理解：不确定性越大，信息熵越大。

2.1.2 条件熵

H(X|Y) = −∑x, yp(x, y)log (p(x|y))

条件熵H(X|Y)表示在移植随机变量Y的条件下，随机变量X的不确定性。

2.1.3 信息增益

信息增益是知道了某个条件后，事件不确定性降低的幅度。信息增益是非对称的，用以度量两种概率分布P和Q的差异，从P到Q的信息增益通常不等于从Q到P的信息增益。

公式：IG(S, A) = H(S) − H(S|A)

表示特征 A 带来的不确定性减少量

2.2 基尼系数

基尼系数可以在样本集中随机抽出两个样本不同类别的概率。

数据集的基尼系数：$Gini(D)=\sum_{k=1}^n\sum_{k\prime\neq k}p_kp_{k\prime}=1-\sum_{k=1}^\mathrm{n}p_k^2$

pk表示第k类样本在数据集中的比例

值域：[0, 0.5]，（二分类时最大值为 0.5，K分类时最大为1-1/K）

特征划分后的加权基尼系数：$Gini(S,A)=\sum_{v=1}^m\frac{|S_v|}{|S|}\cdot Gini(S_v)$

S：特征A取第v个值时对应的子集

$\frac{|S_{v}|}{|S|}$：子集的样本权重

Gini(Sv)： 子集的基尼系数

2.3 增益比

信息增益比较偏好可取值较多的属性，比如我们的样本有一个属性叫序号，每一个样本都具有一个单独的序号，因此使用序号划分后，每个子结点只有一个样本，熵为0。这样的话信息增益最大，算法就会以此属性作为最优划分属性。这显然与我们的意愿不同。因此引申出了增益比的思想。可以说，增益比就是为了矫正信息增益偏好的问题。为了使算法不偏向可取值较多的属性。

$Gain_ratio(D,a)=\frac{Gain(D,a)}{IV(a)}$

其中$IV(a)=-\sum_{\mathrm{i}=1}^V\frac{|D^i|}{|D|}log_2\frac{|D^i|}{|D|}$，是a的固有属性

2.4 均方误差

$\mathrm{MSE}=\frac{1}{n}\sum_{i=1}^{n}(y_{i}-\hat{y}_{i})^{2}$

n：样本数量

yi：第 i 个样本的真实值

ŷi：第 i 个样本的预测值

用于回归问题，衡量预测值和真实值的差异。MSE 越小，表示回归树的预测效果越好。

5.剪枝处理

剪枝处理是防止决策树过拟合的有效手段。剪枝分为“预剪枝”和“后剪枝”。

  1. 预剪枝：在决策树生成过程中，对每个结点在划分前先进性估计，若当前结点的划分不能带来决策树泛化性能提升，则停止划分并将当前结点标记为叶结点。它的位置在每一次生成分支节点前，先判断有没有必要生成，如没有必要，则停止划分。
  2. 后剪枝：先从训练集生成一棵完整的决策树（相当于结束位置），然后自底向上的对非叶结点进行考察，若将该结点对应的子树替换为叶结点能带来决策树泛化性能提升，则将该子树替换为叶结点，相当于将子树剪去。值得注意的是，后剪枝时要用到一个测试数据集合，如果存在某个叶子剪去后能使得在测试集上的准确度或其他测度不降低（不变得更坏），则剪去该叶子。 理论上讲，后剪枝生成的决策树要比预剪枝生成的效果好，但是后剪枝在计算复杂度上比预剪枝高。

Driver,BSP和HAL

Posted on 2025-07-15 Edited on 2025-07-21 In 嵌入式

基本关系如下图所示

1.驱动（Driver）

驱动程序是直接与硬件设备交互的软件组件。它们为操作系统或应用程序提供控制硬件的接口。可以理解为驱动直接与硬件交互。最为底层。

2.板级支持包（BSP, Board Support Package）

是一套针对特定硬件系统的低级软件代码， 其主要作用是抽象硬件细节， 以便操作系统能够顺利运行在特定的硬件平台上。BSP 是高度硬件相关的，为一块板写的 BSP 通常不能直接用在另一块不同的板上，即使它们使用了相同的 MCU/MPU（因为外围电路、引脚连接、时钟源等都可能不同）。

3.硬件抽象层（HAL, Hardware Abstraction Layer）

HAL是介于底层硬件和上层软件之间的一层抽象层，用于隐藏硬件的具体实现细节，提供统一的接口。包含一组通用的API接口，可以理解为定义好的一组标准的函数（如 HAL_UART_Transmit(), HAL_GPIO_WritePin(), HAL_ADC_Start()）来执行常见的硬件操作。

目的是提供统一的、标准化的接口来访问某一类硬件功能（如 UART、I2C、GPIO、ADC、定时器等），屏蔽不同厂商、不同系列、甚至不同架构（如 ARM Cortex-M vs RISC-V）芯片的具体寄存器操作差异。 相对于BSP抽象层次更高，可移植性更高。

以stm32HAL库开发为例，流程如下：

应用程序 │ ↓ 调用统一API HAL 层（如 HAL_UART_Transmit()） │ ↓ 调用板级驱动或直接操作寄存器 BSP 层（如 BSP_UART_Write() 或 直接写 UART->DR 寄存器） │ ↓ 操作物理寄存器 硬件寄存器（如 USART1->DR = data）
