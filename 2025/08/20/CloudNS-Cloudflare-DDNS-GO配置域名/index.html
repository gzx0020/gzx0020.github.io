CloudNS+Cloudflare+DDNS-GO配置域名

Posted on 2025-08-20 Edited on 2025-08-21 Views:

CloudNS获取免费域名

  1. 登录后点击创建区域 -> 免费区域 ，有两种后缀（.com和.net）均可以托管到Cloudflare，任选其一。遇到ip问题可以多更换几个。

  2. 然后往下翻删除掉所有的NS类型的记录

托管至Cloudflare

  1. 账户主页点击加入域

  2. 输入你的域名

  3. 选择free计划

  4. 扫描DNS记录，点击继续，确认以后再添加

  5. 分别复制你已分配的Cloudflare名称服务器（有两个）

  6. 返回CloudNS界面，点击添加新纪录，选择类型为NS，将Cloudflare名称服务器粘贴到指向到一栏

  7. 返回Cloudflare一路继续，最后一步点击立即检查名称服务器，等待邮件提示你已托管成功。

  8. 这一步是我第一次配置时忽略的一步，结果是查看DDNS-GO日志没有日常，但是域名无法解析。

    解决方案是：

    重新返回CloudNS界面，选择A记录，将服务器ip填入指向到一栏，重新尝试，域名即可正确解析：

DDNS-GO配置

  1. 服务商选择Cloudflare，点击小字创建令牌

  2. 跳转至Cloudflare，创建令牌->使用模板，选择你的域名

  3. 创建token并填入

  4. 设置IPv4和IPv6，将你需要解析的域名填入Domains

  5. 保存

Cloudflare Tunnel绕开备案

  1. 开启Cloudflare Tunnel zero trust,选择free

  2. 添加隧道 -> 选择隧道类型 Cloudflared -> 为隧道命名

  3. 安装并运行连接器（以docker为例）

    安装成功后会显示已连接的连接器：

🔧 Cloudflare Tunnel 如何工作与规避备案 Cloudflare Tunnel 的工作原理和规避备案的关键在于：

出站连接：Tunnel 的核心是 cloudflared 守护进程在你的服务器上运行，它会主动与 Cloudflare 的边缘网络建立加密的出站连接。这意味着你的服务器不需要在防火墙上有任何开放的入站端口（包括 80 和 443）。

规避检测：由于连接是由内向外发起的，并且流量被加密，国内运营商通常无法直接检测到你的服务器上正在运行具体的 Web 服务，从而绕过了对未备案域名直接访问服务器端口的拦截机制。

流量路径：用户的访问请求 → Cloudflare 全球网络 → 已建立的加密 Tunnel → 你的服务器（内网服务）。

📝 操作步骤简要指南 安装 cloudflared：在你的服务器上下载并安装 Cloudflare 的守护进程。

bash # 适用于 Linux amd64 的示例 wget https://github.com/cloudflare/cloudflared/releases/latest/download/cloudflared-linux-amd64.deb sudo dpkg -i cloudflared-linux-amd64.deb 验证并创建 Tunnel：

在 Cloudflare Zero Trust 面板中，进入 Networks -> Tunnels。

点击 Create a tunnel，输入一个隧道名称（例如 my-tunnel）。

选择 Debian 操作系统，你会看到一段 cloudflared service install 的命令。在你的服务器上运行这段命令，即可完成隧道的创建和连接。

配置公共主机名：

在隧道配置中，进入 Public Hostnames 标签页。

添加一个公共主机名：

Subdomain: 填写你想要的子域名，例如 www 或 service。

Domain: 选择你托管在 Cloudflare 的域名。

Service: 选择 HTTP 或 HTTPS，并在后面的输入框中填写你的内部服务地址，例如 localhost:8080（如果你本地的 Web 服务运行在 8080 端口）或 http://192.168.1.100:3000。

检查 DNS 记录：配置完成后，Cloudflare 会自动为你的域名创建一条 CNAME 记录，指向 Tunnel 的地址（通常以 .cfargotunnel.com 结尾）。你无需手动修改 DNS 记录。

⚠️ 重要注意事项 政策风险：使用技术手段绕过备案要求，可能违反云服务商的服务条款和相关互联网管理规定。此方案主要用于学习和测试目的，用于生产环境存在风险，请谨慎评估。

性能影响：由于所有流量都需要经过 Cloudflare 的全球网络节点进行中转，访问速度和延迟取决于你服务器到 Cloudflare 入口节点以及用户到 Cloudflare 出口节点的线路质量。对于国内用户访问国内服务器，速度可能不如备案后的直连访问理想。

功能限制：Cloudflare Tunnel 的免费版在一些高级功能上可能存在限制。

非标准端口：部分云厂商可能会检测并封锁除80/443外其他端号的HTTP/HTTPS流量9。

💎 结论与建议 Cloudflare Tunnel 理论上可以用于绕过备案拦截，它通过加密的出站连接隐藏了你的服务器和服务。

如果你的需求是：

快速让一个临时的、小型的、或非商业的网站或应用（例如NAS界面、开发测试项目、个人博客）能够通过域名访问。

服务器本身没有公网IP，或者不想开放任何端口。

那么可以尝试此方案。

如果你的需求是：

运营一个正式的、商业的、希望长期稳定并提供良好用户体验的网站。

主要用户都在国内，对访问速度有较高要求。

那么我强烈建议你： 务必进行备案。这是最合规、最稳定、访问体验最好的方式。备案成功后，你可以继续使用 Cloudflare 的加速和安全服务，只需将橙色云朵点亮即可。
